# Project Governance

Athos Kubernetes follows a lightweight benevolent maintainer model while the
project is in preview.

## Roles

- Maintainer: holds commit access, merges PRs, cuts releases.
- Reviewer: trusted contributor whose review is sufficient for routine PRs.
- Contributor: anyone with a merged PR.

## Decision making

Decisions are made by lazy consensus on PRs and issues. Larger architectural
choices land via an ADR in `docs/adr/`.

## Maintainership

The current maintainer is listed in `.github/CODEOWNERS`. Adding a new
maintainer requires consensus among the existing maintainers.

## Code of Conduct

Contributors agree to abide by [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).
